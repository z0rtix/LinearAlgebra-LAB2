## 1. Подготовка данных

### Генерация основного датасета
Функция `generate_data` создаёт 500 точек с двумя признаками с помощью `make_classification`:
- `n_samples=500` — общее число объектов,
- `n_features=2` — два признака для удобной визуализации,
- `n_redundant=0` — нет бесполезных коррелирующих признаков,
- `n_informative=2` — оба признака информативны,
- `n_clusters_per_class=1` — каждый класс образует одно компактное облако,
- `random_state=42` — фиксация случайности для воспроизводимости.

Данные линейно разделимы, что позволяет перцептрону достичь точности около 0.89.

### Собственные генераторы (доп. задание 1)
- **`generate_linear_separable`** создаёт два гауссовых облака с центрами `[-1,-1]` и `[1,1]`. Используется `np.random.multivariate_normal`. Эта функция генерирует точки из многомерного нормального распределения, которое может иметь скоррелированные признаки (задаётся ковариационной матрицей). В отличие от обычного `np.random.normal`, создающего независимые одномерные величины, `multivariate_normal` способна моделировать зависимость между признаками. У нас ковариационная матрица `[[1,0],[0,1]]` — единичная, что означает независимость признаков и круглую форму облаков. Это идеальный случай для линейного классификатора.
- **`generate_xor`** размещает четыре облака в углах квадрата с добавлением изотропного шума. Данные принципиально не разделимы одной прямой (задача XOR).
- **`generate_circle`** генерирует точки равномерно внутри и снаружи окружности. Граница имеет форму окружности, что также нелинейно.
- **`add_label_noise`** с заданной вероятностью инвертирует метки, имитируя зашумлённые данные.

### Стандартизация
Функция `standardize` выполняет Z-нормализацию **вручную**:
1. Вычисляется среднее `μ` и стандартное отклонение `σ` только на обучающей выборке.
2. Для каждого признака выполняется преобразование `x' = (x - μ) / σ`.
3. Те же `μ` и `σ` применяются к тестовой выборке.

Это приводит признаки к одному масштабу (среднее 0, стандартное отклонение 1), что стабилизирует градиенты и ускоряет сходимость.

### Стратифицированное разбиение
`split_data` делит данные на обучающую (70%) и тестовую (30%) выборки с параметром `stratify=y`. Это гарантирует сохранение пропорций классов в обеих частях, что предотвращает случайное выпадение одного класса из теста.


## 2. Реализация перцептрона (класс `Perceptron`)

Перцептрон — линейная модель для бинарной классификации. Она строит разделяющую гиперплоскость (в 2D — прямую) вида `wᵀx + b = 0`. Вероятность принадлежности к классу 1 вычисляется сигмоидой: `ŷ = σ(wᵀx + b)`.

### Инициализация
- Веса `w` инициализируются малыми случайными значениями (`np.random.randn * 0.01`), чтобы нарушить симметрию и избежать слишком больших начальных шагов.
- Смещение `b = 0`.

### Прямой проход (`forward`)
`z = X·w + b`  
`ŷ = σ(z) = 1 / (1 + e^(-z))`  
Сигмоида преобразует линейный выход в интервал (0,1), интерпретируемый как вероятность.

### Функция потерь (`compute_loss`)
**Бинарная кросс-энтропия** (основная):
```
L = -1/m Σ [y·log(ŷ) + (1-y)·log(1-ŷ)]
```
Где `m` — число примеров. Для численной стабильности `ŷ` обрезается в диапазоне `[1e-15, 1-1e-15]`.

**Hinge loss** (доп. задание 2):
Для меток `y ∈ {-1, +1}`:
```
L = 1/m Σ max(0, 1 - y·(Xw+b))
```
Штрафует, если точка находится внутри отступа (margin < 1).

**L2-регуляризация** добавляет штраф `0.5·λ·||w||²`, чтобы контролировать сложность модели.

### Обучение (`fit`)
Алгоритм мини-батчевого градиентного спуска:
1. Перемешивание данных в начале каждой эпохи.
2. Разбиение на батчи заданного размера.
3. Для каждого батча:
   - Прямой проход → `ŷ`.
   - Ошибка: `δ = ŷ - y` (для кросс-энтропии).
   - Градиенты: `dw = (1/|батч|) X_batchᵀ·δ`, `db = mean(δ)`.
   - При использовании L2-регуляризации: `dw += λ w`.
   - Обновление весов (с возможным моментом):
     - Без момента: `w -= lr·dw`, `b -= lr·db`
     - С моментом: `v = β·v + lr·dw`, `w -= v` (аналогично для `b`).
4. После каждой эпохи вычисляется loss на всей обучающей и валидационной выборке, значения сохраняются для графиков.

### Предсказание (`predict`)
`ŷ = forward(X)`. Класс назначается по порогу 0.5: если `ŷ >= 0.5` → класс 1, иначе 0.

## 5. Особенности реализации

- **Полиморфизм потерь** — один класс поддерживает кросс-энтропию и Hinge loss.
- **Ручные градиенты** — все производные вычислены аналитически, без автограда.
- **Векторизация** — все операции матричные, без циклов по отдельным примерам.
- **Стандартизация вручную** — показывает понимание процесса нормализации.
- **Сохранение графиков** — высокое разрешение (DPI=150), автоматическая перезапись.
- **Чистый код** — без лишних комментариев, единый стиль, строгая структура.
