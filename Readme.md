# Лабораторная работа №1 — Однослойный перцептрон: реализация, обучение и анализ

**Цель работы**
- Реализовать алгоритм обучения однослойного перцептрона **с нуля** (без готовых библиотек глубокого обучения).
- Изучить влияние параметров (скорость обучения, количество эпох, размер батча) на процесс обучения.
- Научиться подготавливать данные для нейронных сетей (нормализация, стратификация).
- Провести анализ полученных результатов и сделать выводы.


## Структура проекта

- **`data_utils.py`** — генерация синтетических данных, ручная Z-стандартизация, стратифицированное разбиение.
- **`perceptron.py`** — класс `Perceptron` (сигмоида, forward, compute_loss, fit с мини-батчами, predict).
- **`experiments.py`** — обучение, визуализация, все эксперименты (обязательные и дополнительные), метрики, кросс-валидация.
- **`main.py`** — точка входа: последовательный запуск всех частей.
- **`results/`** — папка с сохранёнными графиками (создаётся автоматически).
- **`.gitignore`** — исключение служебных файлов из репозитория.
- **`README.md`** — этот отчёт.

## Обязательная часть

### 1. Подготовка данных (`data_utils.py`)
- Данные сгенерированы функцией `make_classification` с параметрами:  
  `n_samples=500, n_features=2, n_redundant=0, n_informative=2, random_state=42, n_clusters_per_class=1`.
- **Ручная Z-стандартизация** (`standardize`): среднее и стандартное отклонение вычисляются на обучающей выборке и применяются к тестовой.
- **Стратифицированное разбиение** 70/30 (`split_data`) с сохранением пропорций классов.

### 2. Реализация перцептрона (`perceptron.py`)
Класс `Perceptron` содержит:
- Инициализацию весов `w` малыми случайными значениями (`N(0, 0.01)`), `b = 0`.
- Сигмоидную функцию активации: `σ(z) = 1 / (1 + exp(-z))`.
- Прямой проход `forward(X)`: вычисление `ŷ = σ(X·w + b)`.
- Функцию потерь `compute_loss` — бинарная кросс-энтропия.
- Метод `fit` с мини-батчами:
  - перемешивание перед каждой эпохой,
  - градиенты вычисляются *вручную* по формулам `∂L/∂w = (1/m) Xᵀ(ŷ - y)`, `∂L/∂b = (1/m) Σ(ŷ - y)`,
  - обновление весов с заданным `lr`,
  - поддержка **момента** и **L2-регуляризации** (опционально),
  - сохранение значений потерь на train/val каждую эпоху.
- Метод `predict(X)` — бинаризация по порогу 0.5.
- **Не используются** PyTorch, TensorFlow, Keras, sklearn.linear_model — только NumPy.

### 3. Обучение и визуализация (`experiments.py`)
- Обучение с гиперпараметрами `η = 0.1, эпох = 100, batch_size = 32`.
- График потерь (train/val) по эпохам.
- Визуализация разделяющей прямой `wᵀx + b = 0` на фоне точек.
- Точность на обучающей и тестовой выборках (baseline accuracy ~0.8867).

### 4. Эксперименты и анализ
Каждый эксперимент — отдельное обучение с фиксацией остальных параметров. Для каждого приведены **графики loss** и **таблицы точностей**.

#### Влияние скорости обучения (`lr`)
*Значения:* `0.001, 0.01, 0.5, 1.0`  
**Вывод:** маленький `lr` замедляет сходимость, большой может привести к снижению точности. Оптимальны умеренные значения.

#### Влияние размера батча (`batch_size`)
*Значения:* `1, 16, 64, 256`  
**Вывод:** слишком маленький батч даёт сильные колебания loss, большой — более гладкую кривую. Точность меняется незначительно.

#### Влияние инициализации весов
*Варианты:* нули, малые случайные (`N(0,0.01)`), большие случайные (`N(0,10)`)  
**Вывод:** нули и малые случайные работают хорошо, большие начальные веса могут замедлить сходимость.


## Дополнительные задания

### Задание 1. Собственный генератор данных
Реализованы функции:
- `generate_linear_separable` — два гауссовых облака,
- `generate_xor` — точки в углах квадрата (проблема XOR),
- `generate_circle` — точки внутри/снаружи окружности,
- `add_label_noise` — добавление шума меток.

**Результаты:** перцептрон успешно разделяет линейные данные (acc ≈ 0.93) и ожидаемо проваливается на XOR (acc ≈ 0.35) и окружности (acc ≈ 0.67), что подтверждает ограничения линейного классификатора.

### Задание 2. Дополнительные функции потерь и регуляризация
- Реализован **Hinge loss** (`loss_type='hinge'`) — точность на тесте 0.92.
- Добавлена **L2-регуляризация** (коэффициент `λ`): с ростом `λ` норма весов уменьшается, модель упрощается, точность остаётся стабильной.

### Задание 3. Метрики качества и анализ ошибок
На тестовой выборке вычислены:
- **Precision** = 0.8537
- **Recall** = 0.9333
- **F1-score** = 0.8917
- **ROC-AUC** = 0.9413 (построена ROC-кривая)
- Ошибочные точки визуализированы: большинство ошибок сосредоточено вблизи разделяющей границы.

### Задание 4. Исследование сходимости градиентного спуска
Реализован **градиентный спуск с моментом** (`β = 0.5, 0.9, 0.99`).  
Импульс ускоряет сходимость, наилучший результат при `β = 0.99` (точность 0.8933).

### Задание 5. Кросс-валидация и подбор гиперпараметров
Реализована **5-фолдная стратифицированная кросс-валидация** для подбора `lr` и `batch_size`.  
**Лучшая пара:** `lr = 0.01, batch_size = 64`.  
Финальная модель обучена на всех тренировочных данных и подтвердила точность на тесте 0.8867.

## Особенности реализации

- **Полиморфизм потерь:** единый класс `Perceptron` поддерживает кросс-энтропию и Hinge loss через параметр `loss_type`.
- **Ручные градиенты:** все производные вычислены аналитически, без автоматического дифференцирования.
- **Векторизация:** все операции над батчами реализованы через `NumPy`.
- **Кросс-валидация:** используется `StratifiedKFold` только для разбиения, не для обучения.
- **Сохранение графиков:** высокое разрешение (DPI=150), перезапись при каждом запуске.
- **Чистый код:** без комментариев, единый стиль, строгая структура.
